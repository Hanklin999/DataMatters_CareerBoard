# Data Matters 使用說明

[開始探索](https://datamatters-hanks-career-board.netlify.app/) · [English Project README](README.md)

Data Matters 是一個給大學生與職涯初期使用者的「資料職涯探索工具」。

它不會替你決定工作，也不是心理測驗。它會根據你偏好的工作方式，整理出較值得優先探索的資料職涯方向，並提供角色差異、常見工作內容、職稱與職缺來源。

---

## 這個網站可以幫你做什麼？

你會得到：

- 一個最接近你工作偏好的資料職涯方向；
- 兩個也值得探索的相近方向；
- 推薦原因；
- 常見工作內容與職稱；
- 不同角色的比較；
- 可進一步查看的職缺來源；
- 接下來可執行的探索建議。

網站目前涵蓋的方向包括：

- 資料分析與商業智慧
- 資料科學
- 機器學習工程
- 資料工程
- 營運研究與最佳化
- 策略與商業分析
- 產品與系統分析
- 金融與風險分析
- 資料治理與品質

## 如何使用？

### 1. 開始測驗

進入首頁後，點選開始探索。

網站會先詢問你目前對資料工作的了解程度。這只用來評估網站是否有幫助，不會影響推薦結果。

### 2. 回答工作偏好

題目會詢問你對不同工作方式的偏好，例如：

- 願不願意投入程式與技術；
- 喜歡明確問題或模糊問題；
- 偏好獨立深入研究或頻繁協作；
- 喜歡分析、預測、建立系統或推動決策；
- 重視穩定交付或開放式探索。

請依照「你比較願意長期做哪一種工作」回答，而不是依照哪個選項看起來比較厲害。

### 3. 查看探索結果

完成後，你會看到：

1. 最接近你的角色；
2. 推薦原因；
3. 可能的真實職稱；
4. 其他相近方向；
5. 工作偏好與技術投入；
6. 相關職缺；
7. 接下來可以做的事。

### 4. 比較角色

對兩個方向仍分不清楚時，可使用角色比較功能。

比較內容包含：

- 每天主要做什麼；
- 常見產出；
- 技術投入；
- 常合作的人；
- 常見職稱；
- 容易被混淆的地方。

### 5. 查看職缺

職缺範例是用來幫助你理解市場上的實際職稱與工作內容。

請注意：

- 職缺可能已下架；
- 公司對相同職稱的定義可能不同；
- 應以外部來源頁面的最新資訊為準；
- 一個職缺同時可能包含多種角色的工作。

## 如何解讀推薦？

推薦結果代表：

> 根據你目前填寫的工作偏好，這些角色較值得優先了解。

它不代表：

- 你只能做這個角色；
- 你已經具備所有技能；
- 這是人格或能力測驗；
- 其他角色不適合你；
- 選擇這個方向一定能找到工作。

「最接近你」與「也值得探索」的差距可能很小。建議不要只看第一名，也要閱讀相近方向。

---

## 為什麼結果可能和你想的不一樣？

常見原因包括：

- 你原本是依照科系或熟悉的職稱選工作；
- 你喜歡某個產業，但未必喜歡該角色的日常工作；
- 你對不同職稱的理解與網站分類不同；
- 你的偏好尚未集中在單一方向；
- 你回答的是「目前會什麼」，而不是「未來想做什麼」。

可嘗試重新作答，並把重點放在工作內容與工作方式。

## 建議的下一步

完成結果後，不需要立刻決定職涯。先做三件事：

### 讀 5 份職缺

搜尋推薦角色的常見職稱，記下重複出現的：

- 工作內容；
- 工具；
- 技能；
- 經驗要求；
- 合作對象。

### 做一個小型作品

作品不必很大。重點是模擬該角色會處理的問題與產出。

例如：

- 分析角色：回答一個決策問題；
- 資料科學角色：建立預測或實驗分析；
- 資料工程角色：建立可重複執行的資料流程；
- 最佳化角色：在限制下提出資源配置方案；
- 產品角色：把需求轉成指標、流程或功能設計。

### 找一位從業者聊聊

用 20 分鐘確認：

- 一天通常在做什麼；
- 最難的工作是什麼；
- 新人最容易誤解什麼；
- 哪些技能是工作後才真正需要；
- 你是否喜歡這種工作節奏。

## 資料與隱私

網站使用匿名產品分析，以了解使用流程並改善題目與介面。

目前原則包括：

- 不需要註冊帳號；
- 不要求姓名、Email 或履歷；
- 使用匿名 session；
- 不做裝置指紋辨識；
- 不儲存完整瀏覽器內容；
- 不把原始答案物件直接傳送到分析資料庫；
- 分析失敗不會影響網站功能。

網站可能記錄的互動包括：

- 是否開始與完成測驗；
- 每一階段是否完成；
- 題目作答時間與是否更改答案；
- 推薦角色；
- 是否查看角色、比較角色或開啟職缺；
- 使用者自願提供的清楚度與符合程度評分；
- 裝置類型、來源網域與活動參數。

---

## 使用限制

Data Matters 是探索工具，不是正式職涯評量。

推薦內容可能受到以下限制：

- 題目與權重仍需持續驗證；
- 不同公司對職稱的定義不同；
- 使用者的偏好會隨經驗改變；
- 自我回報不等於實際能力或長期滿意度；
- 網站無法涵蓋所有產業與特殊職位。

請把結果當成「縮小探索範圍的起點」，而不是最終答案。

---

## 回饋

完成結果後，可在頁面回報：

- 看完後是否更清楚；
- 推薦是否符合你的感覺。

回饋只用於改善題目、推薦邏輯與閱讀體驗。
